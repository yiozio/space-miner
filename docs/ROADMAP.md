# Space Miner 開発ロードマップ / TODO

「ゲームとして成り立っていない」という課題に対する、優先度つきの改善計画。
セッションのコンテキストに依存せず参照できるよう、評価と具体タスクをこのファイルに集約する。
未実装の細目は [`KNOWN_GAPS.md`](KNOWN_GAPS.md)、設計判断は [`adr/`](adr/README.md) を参照。

## 現状の課題

- 報酬が完全に予測可能（小惑星 1 グリッド = 1 資源、固定価格）で**射幸感がない**。
- 短期目標（酒場クエスト）はあるが、**中期・長期の目的＝達成感がない**。
- 採掘・戦闘の**フィードバック演出が乏しい**（ダメージ点滅・画面シェイク・回収表示なし）。

## 基本方針

**「射幸感・達成感の土台」→「レベルデザインの充実」の順**で進める。
報酬を受け取る瞬間が気持ちよくない状態でパーツ種類だけ増やしても、
「良いパーツを買う動機」が弱いまま。土台のループを先に直す方が費用対効果が高い。

レベルデザインも「数」より「意思決定」を重視する。上位互換を並べるだけだと
「金を貯めて順に買う作業」になるため、トレードオフのある選択を作る。

## 評価サマリ

### 当たりグリッド方式（ユーザー指示）

**狙い:** 小惑星グリッドは基本的に資源を落とさず、キラキラ光る「当たりグリッド」からのみ
資源が出る。稀に全グリッドが当たりの小惑星も湧く。

**評価: 採用。射幸感の核になる良い変更。** 以下の効果がある。

- 画面内に常に「あれを狙いたい」という短期目標（光る当たり）が生まれる。
- 採掘が「ただ岩を割る作業」から「当たりを狙う」スキル/判断ゲームになる。
- 光る演出そのものが射幸感のシグナルになり、全当たり小惑星は大当たり（ジャックポット）。

**リスク（要調整・下記タスクでケア）:**

- 当たりが少なすぎると「ハズレ岩を延々割る」徒労感になる → 当たり割合の調整が要。
- AutoAim が最寄りグリッド（≒ハズレ）ばかり撃つと資源が出ない → 当たり優先ロジック。
- ドロップ総量が減るため経済の再調整が必要。

### 改善案全体の評価

- **フィードバック演出（Phase 1）**: 実装が小さい割に体感が激変。最優先。
- **変動報酬（Phase 2）**: 当たりグリッド + レアドロップ。射幸感の本体。
- **経済圧力（Phase 3）**: 修理/給油の有料化・ワープ燃料コスト。「稼ぐ」に意味を与える。
- **解禁構造（Phase 4）**: 達成感（中期目標）。既知ギャップ「ワープ先発見/解禁」に対応。
- **レベルデザイン（Phase 5）**: トレードオフ・地域価格差。土台ができた後に効く。

## Phase 1: フィードバック演出（最優先・小規模）

採掘と戦闘の「手応え」を作る。KNOWN_GAPS の演出項目に対応。

- [ ] 小惑星/海賊の破壊時に小さな画面シェイク（カメラオフセットを数フレーム揺らす）
- [ ] 破壊・命中時に 1〜2 フレームのヒットストップ（更新を一瞬止める）
- [ ] 資源回収時に「+1 鉄」フローティングテキスト（上方向にフェードアウト）
- [ ] 被弾時の自機/海賊の色フラッシュ（KNOWN_GAPS「ダメージ点滅」）
- [ ] 連続破壊で効果音の音程を上げる（手続き合成なので安価。コンボ感）
- [ ] 海賊撃破の爆発エフェクト強化（KNOWN_GAPS「海賊の死亡演出」）

## Phase 2: 変動報酬

### 2a. 当たりグリッド方式（ユーザー指示・着手対象）

実装の前提として、`AsteroidGrid` は全グリッドが Resource/HP を保持したまま、
ドロップの有無だけを当たりフラグで制御する（色・耐久・ミニマップ表示は現状維持）。
初期値の目安: 当たり割合 **20%**（最低 1 個保証）、全当たり小惑星の出現率 **3%**。

- [ ] `AsteroidGrid` に `IsOre bool` を追加し、`Hit`/`HitGrid` で当たりグリッドのみ Pickup 生成（ハズレは破壊しても何も落とさない）
- [ ] `NewAsteroid` に当たり割り当てロジック（割合定数・最低 1 個保証・seed 由来で決定論的）。全当たりフラグ/引数も用意
- [ ] 当たりグリッドのキラキラ描画（`Asteroid.Draw` で明滅。ハズレの岩色と明確に区別。タイトル背景の小惑星にも乗る点に注意）
- [ ] `trySpawnAsteroid` に稀な全当たり小惑星のスポーン（`spawnRng` 使用・確率は data 定数化）
- [ ] 当たり/ハズレ破壊時の音・エフェクト出し分け（当たりは専用 SE + 強めの破片、全当たり撃破は特別演出/画面シェイク）
- [ ] AutoAim が当たりグリッドを優先的に狙う調整（`NearestGridIdx` に当たり優先版。手動射撃とのバランス要判断）
- [ ] 当たり方式に合わせた採掘バランス調整（当たり割合・全当たり率・`MaxAsteroids`・1 当たりのドロップ価値。実プレイで締める）

### 2b. レアドロップ

- [ ] 高価値資源を 1〜2 種追加（貴金属・結晶など。`data_resources.go` / `ResourceWeight` に乗せる）
- [ ] レア資源は当たりグリッドから低確率で出す（当たりグリッド方式と接続）
- [ ] ショップで買えないパーツの低確率ドロップ
- [ ] レア用の光るピックアップ表現 + 専用 SE

## Phase 3: 経済圧力

支出を作り、「稼ぐ」を目的化する。いずれも KNOWN_GAPS 記載の既知ギャップ。

- [ ] 修理（Repair）の有料化（HP 回復量に応じた credits 消費）
- [ ] 給油（Refuel）の有料化
- [ ] ワープの燃料コスト（現状は Warp パーツ装備のみが要件。燃料消費を実装）
- [ ] 燃料切れ時の緊張感（帰投不能リスク等。KNOWN_GAPS「燃料切れ時の挙動」）
- [ ] 撃沈/被撃破時のアセットロス（積載資源の流出など。KNOWN_GAPS）

## Phase 4: 解禁構造（達成感・中期目標）

- [ ] 恒星マップで未到達マップを「???」表示にする（現状は全マップ最初から可視）
- [ ] マップ初到達時の演出 + 報酬（章の節目を作る）
- [ ] 到達/発見によるワープ先アンロック（KNOWN_GAPS「ワープ先発見/解禁」）

## Phase 5: レベルデザイン（土台の後）

- [ ] パーツにトレードオフを持たせる（軽いが弱い/重いが強い、高出力だが燃費が悪い等。重量システム `CanAddWeight` を活用）
- [ ] 資源の地域価格差（マップごとに売値を変える。Cargo と運搬に意味）
- [ ] ステーション間運送ミッション（KNOWN_GAPS。地域価格差と噛み合う）
- [ ] パーツ種類の拡充（上位互換ではなく、用途の異なる選択肢として）

## Phase 6: 長期目標（蓄積の可視化）

- [ ] 図鑑（資源・パーツ・撃破した海賊パターン）
- [ ] 統計画面（Playtime など既存メタを活用）
- [ ] ステーション評判ランク（`VisitedStations` を下地に）

## 推奨着手順

1. **Phase 1**（演出）— 今日規模で体感が変わる。
2. **Phase 2a**（当たりグリッド）— ユーザー指示の着手対象。詳細タスクは上記。
3. **Phase 2b**（レアドロップ）→ **Phase 3**（経済圧力）。
4. **Phase 4**（解禁）→ **Phase 5/6**。

Phase 1〜2 だけでも「成り立っていない」感はかなり解消する見込み。
