# 0013. セーブシステム

## Status
Accepted

## Context
- セーブを 3 スロット持ちたい（Continue / Load 用にスロット選択 UI）。
- スロット選択時には保存日時 / 累計プレイ時間 / 所持金 / 宙域名を表示したい。
- 保存中のクラッシュで既存データが破壊されないこと。
- パーツや派生ステータス（MaxHP 等）はパーツ構成から再計算できるので、
  生データを冗長に保存しない。

## Decision

### スロットとファイル
- 3 スロット (`SlotCount = 3`)、`save_1.json` / `save_2.json` / `save_3.json`。
- 配置: `os.UserConfigDir()/space-miner/`
  - macOS: `~/Library/Application Support/space-miner/`
  - Linux: `$XDG_CONFIG_HOME/space-miner/` または `~/.config/space-miner/`
  - Windows: `%AppData%\space-miner\`
- `Save` は `*.json.tmp` に書き出して `os.Rename` で原子的に置換。

### スキーマ
- `State` : `Version` + `Meta` + `Player`
- `Meta` : `SavedAt` / `Playtime` / `Credits` / `MapName`（スロット選択 UI 用）
- `PlayerState` : 配置パーツ（`DefID, GX, GY, Rotation`）、位置・向き、HP / Shield / Fuel、
  Credits、Inventory、PartsInventory、VisitedStations、Tavern、PiratesKilledByMap
- `Rotation` は `omitempty` で旧セーブ互換（未設定 = 0）。

### ロード復元
- 復元時に `Player.OnPartsChanged()` で派生ステータス（MaxHP / MaxFuel / MaxShieldHP / MaxCargo）を
  再計算し、保存値（HP/Shield/Fuel）を新しい上限内にクランプする。
- 速度は 0 で復元（ロード直後は静止）。
- 過去のクエスト掲示板・撃破数・訪問ステーションを復元することで、
  Bounty 進捗とダイアログ既読状態が継続する。

### UI 連携
- タイトル画面: `Continue` は `LatestSlot()`（最新の SavedAt）を直接ロード、
  `Load` は `SaveSlotPicker` で選ぶ。
- メニュー画面: `Save` / `Load` 共に `SaveSlotPicker` を Push。
  既存スロットへの上書きは確認モーダルを挟む。
- スロットカードに `日時 / 経過時間 / CR n / 宙域名` を表示。

## Consequences

- 派生値を保存しないことで、後からバランス調整しても旧セーブが破綻しにくい
  （部品レジストリと整合性さえ取れば再計算で正常化する）。
- 一方、`PartID` の数値割当が変わると旧セーブが壊れる。
  新パーツは末尾追加（既存値を再利用しない）規約とする。
- `time.Time` は JSON で文字列化（RFC3339）。スロット選択 UI でローカルタイムに変換。
- セーブにワープ中状態や戦闘の刹那情報は入らない（小惑星・弾・海賊・ピックアップ等は
  ロード時に再生成）。
