# 0002. テーマシステム

## Status
Accepted

## Context
レトロベクター風 UI（暗い背景 + 単色ライン）を全画面で統一したい。
プレイヤーが背景色（雰囲気）を切り替えられる設定を提供したい一方で、
描画コードが色を直書きするとテーマ切り替えに耐えられない。

## Decision

- `internal/ui/theme.go` の `Theme` を 3 色プリセットで構成: `Background` / `Line` / `LineDim`。
- プリセット: `Black`（緑系）/ `Navy`（シアン系）/ `DarkGreen`（琥珀系）。
- 描画コードは `theme.Line` / `theme.LineDim` / `theme.Background` を必ず参照し、
  色を直書きしない。
- 例外: 機能的に決まっている色（資源色、海賊弾の赤、レーザービームの琥珀、
  HP バーの赤など）はテーマ非依存で固定し、ゲーム側のシグナルとして使う。
- `Director.Theme()` で取得し、`Director.SetTheme()` で全シーンに反映。
- 設定画面でプリセットを切り替えると即時反映、永続化する。

## Consequences

- 1 行の差し替えで全画面の配色が変わる。
- 機能色（敵 / シールド / HP / 弾）はテーマと独立に視認性を保てる。
- 一方、機能色とテーマ色が衝突しないよう、新規プリセット追加時には機能色との
  コントラストを確認する必要がある。
- 個別の色（背景だけ・ライン色だけ）の単独切り替えは許可しない。
  常にプリセット単位で切り替える。

## 関連
- `internal/ui/theme.go`
- `internal/ui/preferences.go` で設定永続化
