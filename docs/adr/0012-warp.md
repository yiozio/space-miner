# 0012. 恒星マップとワープ

## Status
Accepted

## Context
- FullMap を跨いで移動するために「ワープ」機構が必要。
- ワープ先は恒星系内の他 FullMap（または衛星）から選ぶ UI が要る。
- 恒星マップで見た方向と、ワープ後に自機が向く方向が一致しないと混乱する。

## Decision

### 座標統一
- 恒星マップは恒星を世界座標 (0, 0)、各 FullMap を `m.CX, m.CY` で配置する
  ([ADR 0003](./0003-world-hierarchy.md))。
- これによりレイアウトと実際の世界座標差が同じ向きとなり、
  ワープ後の機首角度と恒星マップ上の方向が自動的に一致する。

### 恒星マップシーン (`StarMap`)
- `N` キーで開く。恒星 + 各 FullMap の Body を矩形領域内に自動スケールしてレイアウト。
- 衛星は `ParentName` で親惑星と結ぶ補助線を描く。
- カーソルで FullMap を選択、`Enter` で確定 → コールバック (`onWarp`) 経由で
  `Exploration.startWarp` を呼ぶ。
- Warp パーツ未搭載 (`!Player.HasWarpDrive()`) の場合は閲覧のみ可、確定無効。
  画面下に `WARP DRIVE NOT INSTALLED` 表示。

### ワープ実行とアニメ
- `startWarp(dest)`：
  - 機首角度を `atan2(dest.CY - lastMap.CY, dest.CX - lastMap.CX)` で目的地方向に向ける
    （区画外なら自機座標差で代替）。
  - 速度 0、`warpTimer = warpDuration (=90)`、`warpDest = dest`。
- `tickWarp()` が毎フレーム時間を進める。`warpTimer == 45`（中点）で実テレポート：
  `player.X/Y = dest.CX/Y`、`lastMap = dest`、小惑星・ピックアップ・弾・海賊・着弾エフェクト・
  ビーム・自動照準ターゲットをすべてクリア（前マップの残骸を持ち越さない）。
- `drawWarpOverlay`：機首方向にライン群を流す（`forward = (cos, sin)` × `streakLen` の方向）。
  ホワイトアウト透明度は `sin(progress · π)` で中点ピーク。HUD はワープ中非表示。

## Consequences

- 「マップを見る」「方向を選ぶ」「ワープする」が直感的に整合する。
- ワープが「世界の切り替え」ではなく「同じ World の別の場所への跳躍」として処理されるため、
  `lastMap` の更新と一時状態のリセット以外は通常 Update に戻るだけで済む。
- 区画外でも忍耐強く航行すれば別 FullMap に到達できる（パンチ抜きの探索が可能）。
- セーブには Warp 状態を含めない（ワープ中の保存は想定しない）。
