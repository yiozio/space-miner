# プロジェクト概要
2D宇宙採掘ゲーム「Space Miner」の開発。

# ビルド確認
ビルド確認には必ず `go vet ./...` を使用すること

# フォーマット
実装が終わったら必ず `gofmt -w .` を実行すること

# 技術スタック
- 言語: Go 1.22+
- ゲームエンジン: ebitengine v2

# コーディング規約
- Effective Goに倣うこと
- コメントは日本語で行うこと
- パッケージ構成はクリーンアーキテクチャを意識する
- ゲームの状態管理はステートマシンパターンを使用する
- データと実装を分離する規約: バランス・コンテンツ系のデータは `data_*.go` ファイルに集約する（[ADR 0005](docs/adr/0005-part-defs.md)）

# 参照すべきドキュメント
- `docs/adr/`: アーキテクチャ決定記録（ADR）。設計判断の文脈と帰結。索引は [`docs/adr/README.md`](docs/adr/README.md)
- `docs/KNOWN_GAPS.md`: 既知の未実装項目・将来課題。新機能を着手するときの起点

実装の細部はコードを正とする。ADR は why と前提の記録に徹する。
